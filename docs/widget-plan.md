# Widget Plan

The widget family is the 30-row spec in `AI_Labs_Widget_Specification - Rich Chat Widgets.csv`. Every widget below ‚Äî built or pending ‚Äî follows **`docs/widget-conventions.md`** (the rule book). Non-negotiables for every entry:

- ¬ß0 two-pass workflow ‚Äî Pass 1 structure, then Pass 2 elevation with `/frontend-design`.
- ¬ß0.1 zero hardcoded values ‚Äî every px / rem / hex resolves to a Nexus token.
- ¬ß1 card shell ‚Äî symmetric `space-200` padding, `grey-10` border, hover-only shadow, slot-owned width.
- ¬ß2 header chrome ‚Äî 36√ó36 brand-tinted icon badge, title at `font-size-400`, description at `grey-60`.
- ¬ß5 primary CTA ‚Äî `Button variant="primary"`, full width, accent override `--color-action-primary: var(--brand-60)`, sentence-case verb label.
- ¬ß13 color conventions ‚Äî brand-60 accent, `grey-10 / 20 / 30 / 50 / 60 / 80 / 90` neutral ladder, `--color-text-success` / `--color-text-error` / `yellow-60` for status.
- ¬ß15 Lucide icons only.
- ¬ß16 three motion curves ‚Äî state transitions, rise-up entry, springy bounce. No fourth curve.
- ¬ß17 five-touchpoint registration ‚Äî component file, `registry.js`, `widgetSchemas.js`, `mockBot.js`, `WidgetPalette.jsx` icon map.

If `/frontend-design` ever proposes something that conflicts with the conventions doc, **the doc wins**. Anti-pattern audit (¬ß15, ¬ß18) must pass zero before shipping.
